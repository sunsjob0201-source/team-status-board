# 共同作業の約束

このリポジトリは、安心してレビューを練習するための教材です。分からないことがあっても大丈夫です。

## 作業の流れ

1. Issueを読み、自分を担当者（Assignee）に設定します。
2. `feature/issue-番号-短い名前` という名前でブランチを作ります。例：`feature/issue-12-tanaka`
3. `main` ブランチへ直接コミットせず、作業ブランチで変更します。
4. コミットの前に「Changes」タブで差分を確認します。
5. 内容が分かるコミットメッセージを書きます。例：`feat: 田中のメンバーカードを追加`
6. Pull Requestを作り、別の参加者へレビューを依頼します。

## レビューの約束

- 人ではなく、変更内容を対象にコメントします。
- 「間違いです」だけで終わらせず、理由や修正例を添えます。
- 良かった点も具体的に伝えます。
- 分からない点は、決めつけず質問として書きます。

コメント例：

> `status` は定義された4種類から選ぶ必要があります。「作業中」へ変更すると自動チェックが通りそうです。
