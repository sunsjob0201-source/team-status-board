# 確認方法

このプロジェクトは追加パッケージや専用テストフレームワークを使いません。Node.jsの標準機能による自動検証と、ブラウザでの手動確認を組み合わせます。

## 自動確認

リポジトリのルートで次を実行します。

```text
node scripts/validate.mjs
node scripts/build.mjs
```

正常時は、どちらも `✓` から始まるメッセージを表示して終了コード0で完了します。`dist/index.html`、`dist/assets/style.css`、`dist/assets/app.js`、`dist/data.json` が生成されます。

## エラー確認

演習用ブランチでのみ行い、確認後は不正ファイルをコミットせず削除してください。

1. `members/invalid.json` を作ります。
2. 内容を `{ "name": "テスト" }` にします。
3. `node scripts/validate.mjs` を実行します。
4. 終了コードが1になり、`members/invalid.json` と不足項目が日本語で表示されることを確認します。
5. ファイル名を `Invalid_Name.json` に変え、ファイル名ルールの説明も表示されることを確認します。
6. 不正ファイルを削除し、検証が再び成功することを確認します。

追加で確認する代表例：

- JSON末尾のカンマやダブルクォートを欠かすと構文エラーになる
- `status` に `対応中` を入れると許可された4種類が表示される
- 同じ `name` のファイルを2つ置くと、両方のファイル名が表示される
- `message` を101文字にすると現在の文字数が表示される
- `members/` を空にしても検証とビルドは成功する

## ブラウザでの手動確認

ビルド後、次を実行して `http://127.0.0.1:4173` を開きます。

```text
node scripts/serve.mjs
```

- [ ] お知らせの日本語が文字化けせず表示される
- [ ] 登録メンバー数が2人、完了人数が0人と表示される
- [ ] カードが名前の昇順で表示される
- [ ] 名前、担当、状態、ひとことが各カードにある
- [ ] 状態バッジは色だけでなく文字でも区別できる
- [ ] PC幅ではカードが複数列になる
- [ ] 640px以下ではカードと集計が1列になる
- [ ] 長い名前やひとことがカード外へはみ出さない
- [ ] Tabキーで「メインコンテンツへ移動」にフォーカスできる
- [ ] 開発者ツールのConsoleにエラーがない

## 空状態と取得エラー

空状態は一時ブランチで `members/*.json` を別の場所へ退避してビルドし、「まだメンバーが登録されていません」が表示されることを確認します。

取得エラーは、開発者ツールのNetworkで `data.json` をブロックするか、テスト用に名前を変更して再読み込みします。「データを表示できませんでした」と再読み込みの案内が表示されることを確認します。確認後は必ず元へ戻します。
